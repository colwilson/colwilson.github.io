This series looks at how to configure a brand new Feathers service to use three different LevelDB backends: a memory backed one, and in part 2 a file backed one and a DynamoDB backed one.

Feathers is an open source web framework for building modern real-time applications. I’ll look
at the real-time bit in a later post, but for now let’s just say it allows you to create an API layer over your databases and data stores.

LevelDB is a fast key-value storage library written at Google that provides an ordered mapping from string keys to string values. You can pretty much use any data store you want.
